## 2024-05-27 - GPU Transforms for Continuous Animation in Framer Motion
**Learning:** Continuous animations of element positions using Framer Motion (such as custom cursors tracking pointer movement) should never use layout-bound CSS properties like `left` and `top`. Updating these properties on every frame causes expensive CPU layout recalculations and repaints, leading to dropped frames and jank. Instead, passing `x` and `y` props to Framer Motion leverages GPU-accelerated CSS transforms (`translate`), which bypasses layout and significantly improves performance. Additionally, because Framer Motion manages and overrides the inline `transform` CSS property entirely, centering the element using standard CSS `transform: translate(-50%, -50%)` will break.
**Action:** When animating position continuously with Framer Motion, always use `x` and `y` for translation. To offset or center the element, use negative margins (e.g., `margin: -[height/2]px 0 0 -[width/2]px`) instead of CSS transforms to avoid conflicts with Framer Motion's internal transform management.
