## 2025-02-28 - GPU Accelerated Custom Cursor and Framer Motion Transform Overrides
**Learning:** When continuously animating element positions using Framer Motion (like a custom cursor following a pointer), using `left`/`top` CSS properties forces the browser to perform heavy CPU layout recalculations on every single frame, causing significant performance overhead. Instead, using `x` and `y` maps these to GPU-accelerated `transform: translate3d` CSS, providing much smoother frame rates. Additionally, when using `x` and `y`, Framer Motion completely overrides any inline `transform` property set in CSS (such as `transform: translate(-50%, -50%)`).
**Action:** When animating positional elements per-frame with Framer Motion, always use `x`/`y` for GPU acceleration instead of `left`/`top`. For centering these elements, use negative margins (e.g. `margin: -16px 0 0 -16px` for a 32x32px element) rather than CSS `transform: translate` to prevent Framer Motion from overwriting the layout offsets.
