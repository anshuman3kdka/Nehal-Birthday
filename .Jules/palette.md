## 2025-05-29 - [Animated Navigation Accessibility & UX]
**Learning:** When using Framer Motion `layoutId` pills for animated navigation tabs, developers often rely on visual cues alone to indicate the active state, which causes accessibility issues. By replacing inline text colors with conditional Tailwind classes, we also enable better hover/focus states (`hover:text-` and `focus-visible:ring-2`) and interactive polish (`active:scale-95`). Combining this with standard HTML accessibility attributes like `aria-current="page"` creates a robust, inclusive component.
**Action:** Always pair animated interactive elements (e.g., Framer Motion layoutId pills or animated links) with standard HTML accessibility attributes (like `aria-current="page"` or `aria-label`) and ensure visible focus states for keyboard users (e.g., `focus-visible:ring-2`). Leverage Tailwind conditional classes over inline styles for text colors to allow hover/active interactions.
