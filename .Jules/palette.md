## 2024-03-24 - Accessible Floating Navigation
**Learning:** Custom floating navigation bars (especially those built with motion libraries like Framer Motion) often lack basic semantic accessibility out-of-the-box. Without `aria-label` on the `<nav>`, screen readers cannot quickly locate the main navigation. Furthermore, without `aria-current="page"`, visually active states (like pill highlights) are not conveyed to non-visual users. Finally, keyboard navigation requires visible focus indicators (`focus-visible:ring`), which are frequently overlooked in purely mouse-driven designs.
**Action:** When implementing custom animated navigation components, always verify semantic wrappers (`<nav>`), state communication (`aria-current`), and keyboard focus states (`focus-visible`) as a baseline standard.
